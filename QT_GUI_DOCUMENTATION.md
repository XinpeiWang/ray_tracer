# Qt GUI - Complete Implementation Guide

## ✅ What Was Built

A **professional Qt6-based GUI** for the Ray Tracer with:
- Modern dark theme (Discord-inspired purple/blue color palette)
- Tabbed interface (Basic & Advanced settings)
- CPU/GPU render mode selection
- Quality presets (Preview/Good/Best/Custom)
- Resolution picker (800x800, 1200x1200, 1920x1080, 2048x2048)
- Output path browser
- Real-time progress bar
- Render status display with total time
- Cross-platform foundation (currently Windows, can extend to Linux/Mac)

## 🔨 Building the Qt GUI

### Prerequisites
- Qt 6.11.1 (or later 6.x) with MinGW 64-bit
- MinGW 13.1.0 tools (installed with Qt)

### Build Steps

**Option 1: Manual Build (What We Did)**
```powershell
# Set up environment
$env:Path = "C:\Qt\Tools\mingw1310_64\bin;C:\Qt\6.11.1\mingw_64\bin;$env:Path"

# Generate Makefiles
cd qt_gui
mkdir build
cd build
qmake.exe ..\RayTracerGUI.pro CONFIG+=release

# Compile
mingw32-make.exe -j8

# Deploy Qt DLLs
cd ..\..\RayTracer_Package
windeployqt.exe RayTracerGUI.exe --no-translations
```

**Option 2: Using Qt Creator (Easier)**
1. Open `qt_gui/RayTracerGUI.pro` in Qt Creator
2. Select "MinGW 64-bit" kit
3. Click Build → Build Project
4. Run windeployqt on the output exe

## 🔌 Integration with Render Backend

The Qt GUI launches the ray tracer as a **separate process**:

```cpp
// Command format
ray_tracer.exe --gpu --width 800 --height 800 --samples 100 --depth 50
ray_tracer.exe --cpu --width 1920 --height 1080 --samples 10 --depth 50
```

### How It Works
1. User clicks "Start Render"
2. `RenderThread` spawns `ray_tracer.exe` with appropriate flags
3. Captures stdout/stderr for progress parsing
4. Parses progress percentage (looks for "X%" in output)
5. Updates progress bar in real-time
6. On completion, shows total render time

### Important Note
**The current implementation assumes `ray_tracer.exe` exists in the same directory as `RayTracerGUI.exe` and accepts command-line arguments.** You'll need to:

1. **Update the console launcher** (`launcher/main.cpp`) to accept CLI args:
   ```cpp
   // Parse --gpu, --cpu, --width, --height, --samples, --depth flags
   ```

2. **Or**: Directly link CPU/GPU render APIs into the Qt GUI:
   ```cpp
   // Instead of QProcess, call:
   extern "C" int cpu_render_main(int width, int height, int samples, int depth);
   extern "C" int gpu_render_main(int width, int height, int samples, int depth);
   ```

## 🐛 Troubleshooting

### "Application failed to start"
- **Cause**: Missing Qt DLLs or plugin directories
- **Fix**: Run `windeployqt RayTracerGUI.exe` again

### "Cannot find ray_tracer.exe"
- **Cause**: Console launcher not in same directory
- **Fix**: Copy `ray_tracer.exe` to `RayTracer_Package\`, or integrate render APIs directly

### GUI doesn't show progress
- **Cause**: Console launcher doesn't print progress to stdout
- **Fix**: Add progress output like `printf("%d%%\n", percentage);` in render loop

### Dark theme not applied
- **Cause**: Fusion style not available
- **Fix**: Qt should include Fusion by default; check Qt installation

### Progress Parsing
```cpp
QRegularExpression progressRegex("(\\d+)%");
QRegularExpressionMatch match = progressRegex.match(output);
if (match.hasMatch()) {
	int progress = match.captured(1).toInt();
	emit progressUpdate(progress);
}
```
